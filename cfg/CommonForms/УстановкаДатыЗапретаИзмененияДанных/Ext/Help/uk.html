Установка даты запрета изменения данных

 

В конфигурации поддерживается возможность установки ограничения изменения данных ранее определенной даты (далее «дата запрета»).

Контроль ограничения изменения данных выполняется для:

  • всех документов при проведении, распроведении, отмене проведения;
  • наборов записей периодических регистров сведений, с независимым от регистратора режимом записи;
  • наборов записей регистров накопления, при этом контроль выполняется во всех случаях изменения записей регистров, включая программное.

При этом для документов проверяемым значением является дата этого документа, для наборов записей - период каждой из записей данного набора.

При установке даты запрета следует учитывать, что под действие данного ограничения попадают данные, период действия которых ограничен концом даты запрета. То есть при установке значения 31.12.2007 изменение данных будет возможно только начиная с 01.01.2008 00:00:00 включительно.

Дата запрета устанавливается в разрезе ролей пользователей. В случае если пользователю одновременно доступно несколько ролей, в качестве ограничивающей даты берется минимальная из дат, установленных для этих ролей.

Ограничение по дате запрета не распространяется на пользователей, которым доступна роль «Полные права».

Различаются понятия «основной даты запрета» и «даты ограничения изменения регламентной информации». Соответственно, в область действия основной даты попадают управленческие данные, вторая дата ограничивает изменение регламентной информации, раздельно для каждой из организаций. При этом, если дата запрета для регламентной информации не задана, при контроле изменения этой информации будет применяться основная дата.  Если одновременно не задано значение и для основной даты и для регламентной - контроль изменения данных не будет задействован.

«Основная дата запрета» устанавливается на закладке «Основная дата» формы. Отображение заданных значений в разрезе ролей пользователей организовано в виде дерева, что позволяет задать значение как для всех ролей одновременно (строка дерева «Все роли»), так и для каждой роли в отдельности.

Установка ограничения на изменение регламентной информации выполняется на закладке «Ограничение регламентной информации». Поскольку данное ограничение является расширением механизма контроля, по умолчанию список организаций не заполнен (подразумевается, что запрет на изменение ограничивается основной датой для всех организаций одновременно). В случае необходимости установки даты запрета отдельно для конкретной организации, данную организацию необходимо добавить в список. После чего, в появившемся дереве возможно указание даты запрета как в целом для всей организации, так и для каждой роли в отдельности. В случае, когда необходимость указания отдельной даты запрета для организации пропала, соответствующую ветвь дерева можно удалить из списка.

При изменении (добавлении) документов, содержащих как управленческую, так и регламентную информацию, выполнение операции разрешается в случае когда дата документа удовлетворяет одновременно требованиям как основой даты запрета, так и регламентной.

Примеры использования.

Пример 1. Указана основная дата запрета (01.01.2004), ограничение регламентной информации не задействовано.

В этом случае любое изменение как документов, так и отдельных записей регистров с датой ранее 01.01.2004 не доступно.

 

Пример 2. Указана основная дата запрета (01.01.2004), для организации «Торговый дом «Комплексный» отдельно указана дата 01.06.2004.

Изменение данных управленческого учета доступно только после 01.01.2004. Изменение регламентных данных по организации «Торговый дом «Комплексный» только после 01.06.2004. Например, документ «Поступление товаров и услуг», с установленными флагами «Отражать в упр. учете» и «Отражать в бух. учете» и датой 01.03.2004 нельзя будет провести по организации «Торговый дом «Комплексный», но возможно по любой другой. Также его проведение будет возможно, в случае снятия флажка «Отражать в бух. учете».
